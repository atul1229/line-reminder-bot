# Project Vision

## Vision

Work Assistant 是一個以 LINE 為核心入口的個人工作管理系統。

它的目的不是增加新的工具，而是整合每天分散在不同平台的工作資訊，降低整理資訊與切換工具的成本。

透過提醒、Google Calendar、AI 摘要與工作資訊整合，使用者每天只需要打開 LINE，就能快速掌握今天的重要行程、待辦事項與工作重點，將更多時間投入真正重要的工作。

Work Assistant 希望成為每天工作的第一個入口，也是一天結束時回顧工作的最後一個入口。

---

# Problem Statement

目前日常工作資訊分散於多個平台，例如：

- LINE
- Gmail
- Google Calendar
- ChatGPT
- Chrome
- 個人筆記

每天開始工作前，都需要花費時間重新整理今天的工作內容與優先順序。

這種資訊分散造成：

- 容易遺漏主任或同事交辦事項
- 難以快速掌握今日工作重點
- 工作資訊需要重複查詢
- 每天花費大量時間切換不同工具

Work Assistant 的目標，就是解決這個問題。

---

# Design Philosophy

Work Assistant 不追求功能最多，而是追求每天都值得使用。

每新增一項功能，都必須符合以下原則：

- 能減少資訊整理成本
- 能減少 App 之間的切換
- 能降低遺漏重要事項的機率
- 能真正提升每天的工作效率

如果一項功能無法達成上述目標，即使技術上可以實作，也不列入優先開發。

---

# Core Objectives

第一階段目標：

建立一個可靠且穩定的工作提醒系統。

包含：

- 建立提醒
- 自動提醒
- 今日提醒查詢
- Google Calendar 整合

第二階段目標：

建立每日工作 Dashboard。

每天早上即可快速掌握：

- 今日行程
- 今日待辦
- 即將到期事項
- AI 今日建議

第三階段目標：

導入 AI，讓系統不只是記錄資訊，而是協助整理資訊。

例如：

- 每日摘要
- 每週工作摘要
- 會議摘要
- 工作建議
- 自動整理待辦事項

---

# Target User

目前系統定位為 Personal Productivity Tool。

主要服務對象為系統開發者本人。

系統將依照個人工作流程持續調整，而非以多人使用或商業化為主要目標。

---

# Success Criteria

本專案成功的標準不是完成多少功能，而是是否真正融入每天的工作流程。

成功代表：

- 每天都會打開 LINE 使用
- 每天都能快速掌握今日工作
- 不再因資訊分散而遺漏重要事項
- 降低工作管理成本
- 節省每天整理資訊的時間

---

# Design Decision

本專案曾考慮定位為：

- Reminder Bot
- LINE AI Bot
- Chat Assistant

最後決定定位為：

**Work Assistant**

原因是：

提醒、Google Calendar、AI 都只是功能。

真正要解決的問題，是每天工作資訊分散、需要反覆整理與切換工具。

因此，Work Assistant 的定位不是聊天機器人，而是一個以 LINE 為核心入口的工作資訊中樞（Work Hub）。

---

# Product Principle

> 好的工具，不是增加新的工作，而是減少原本的工作。

Work Assistant 的所有功能，都應圍繞這個原則設計。

每增加一項功能，都必須回答：

「它是否真正讓使用者節省時間？」

如果答案是否定的，就不應列入產品規劃。
